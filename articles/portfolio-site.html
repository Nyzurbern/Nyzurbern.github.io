Hrithik's Portfolio Site

Overview

This is my personal portfolio website where I showcase my projects, skills, and experiences. It's built with HTML & CSS and is designed to be responsive and user-friendly. This article is about how I learnt web development and built this website from scratch, including the challenges I faced and the solutions I implemented.

What was used

  • Language(s): HTML, CSS
  • Framework(s): None
  • Tools: VS Code, GitHub, Figma
  • Platform: Deployed on the web with GitHub Pages

Learning Web Development

I started my journey in web development by learning the fundamentals of HTML and CSS. I did so using a free website called freeCodeCamp, which provided me with interactive lessons and projects to practice my skills. Development Process

Fast forward to June 2026, I needed to a way to showcase my work and achievements in an organsied manner and a porfolio site was the perfect way to do it. So, the development of this portfolio site began with planning and designing the overall structure and user interface. Now I already had a portfolio site URL that was just sitting on some preset code so I started by modifying the basic HTML structure and then styled it with CSS to achieve the desired look and feel. It took many iterations before I was satisfied with things like the colour palette and layout of the first sections in the about section of the site.

Then I wrote the story for the about section, detailing my background, skills, and experiences. After that, I focused on creating an article system that allows me to add cards linking to each article, as well as a page to display all of them so as not to clutter the main page. Now, because I don't know how to write JavaScript as of writing this, I had to rely on HTML and CSS for the interactive elements. This meant manually writing and adding media to every single article and card manually in each individual file. Moreover, some article cards were duplicates of each other from different pages so I had to make sure when I updated one, the other was also updated.

Next, I added a contact section with links to my social media profiles and email address, allowing visitors to get in touch with me easily.

At this point, the site design was done but I was still in the process of writing content for the articles. Motivated to test the site on mobile devices though, I made sure any links that didn't work yet were disabled and pushed the site to production. Immediately, I noticed several issues that needed to be addressed, from tiny elements to weird sizing of text that made the site look awful on smaller devices. This is detailed on further in the next section.

Challenges

One of the main challenges was ensuring the site was responsive and looked good on all device sizes as some things looked different on various screens. To solve this, I utilized CSS's media queries to implement elements that adapt to different screen sizes. Another one was my lack of JavaScript knowledge, which limited the interactivity of the site. I had to rely on HTML and CSS for interactive elements. Additionally, I faced challenges in organizing and managing the content for the articles, as each article had to be manually added and updated in multiple files.

What I learned

I learned many things about web development, including HTML, CSS, and design principles. Sure, the website may not have the flashiest design especially for one aimed at showcasing tech skills in a portfolio. But it serves its purpose of allowing people to learn about my work and experiences, and if it can do that effectively, then I've succeeded.

Trivia

At one point, I wanted to add a really large video to one of my articles, but when I went to push my changes, it wouldn't let me. Turns out github pages has a limit of 100MB for files, and the video I was trying to add was ~130MB. So I had to upload it to youtube first and embed it instead. Now, because this was my first time trying to revert a commit, I faced difficulties understanding what was going on and deleted all the work I had commited including an article I had just spent an hour writing 😩. Future Improvements
