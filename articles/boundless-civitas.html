Boundless Civitas: Build, Manage, Flourish

Preface

Boundless Civitas is a roblox game that's still in development and is nowhere close to release yet. This article is a preface to the game, and will be updated as the game's development progresses.

Overview

Boundless Civitas is a city-state building and management game. Players can construct buildings, manage resources, and make decisions that will affect the growth and development of their city-state. The game will feature a dynamic economy, simulated relations between other city-states, AI NPCs and more allowing players to experience the challenges and rewards of managing a city-state.

What's used

  • Language(s): Luau
  • Tools: Roblox Studio, GitHub, Google Docs
  • Platform: Roblox

Why I started this project

I've always been fascinated by city-building games and the complex underlying systems that drive their mechanics. Moreover, I am also passsionate about urban planning issues in Singapore which is a city-state. As such, I've really yearned for a sandbox where I could experiment with different street designs and building layouts while dealing with the limitations of being a country and a city at the same time. People often say that certain urban planning solutions like narrower streets won't work in Singapore because "it's different" and I wanted to prove them wrong.

I had also been doing an online course on Roblox game development on Udemy and I saw this as the perfect opportunity to apply what I had learned. Thus, Boundless Civitas was born, with my aim being to create a unique experience that combines the strategic depth of traditional city-builders with the social dynamics of a city-state.

Development Journey

The development of Boundless Civitas started off around May of last year and despite having technically been in development for over a year, the game is still in its early stages. The initial focus was on creating a solid foundation for the game's mechanics, including the data saving & loading system, the camera logic and game UI. I started off with some very basic main menu UI, to try to prototype the game and built the game from there.

Before making any gameplay, I had to design the core datasaving and loading system, so that player progress could be preserved across sessions. As a result, I also had to decide how city-states were going to be structured as future provision for other things like building positions to be saved too. I initially used the default data saving system provided by Roblox, but I soon realized that I needed a more robust solution. So I use a module known as ProfileStore which provides me additional featues like preventing multiple instances of the same player joining.

I also use a module loader system meaning all my game scripts are technically just modules with Init and Start functions that get run by the module loader. This helps me keep the game organized and maintainable. An example of an initalise function used in the game for the main menu is shown below.

I wasn't satisfied with the initial main menu, so I eventually changed it to a more polished version, and added the ability to create new city-states with attributes like name, difficulty & biome. I eventually plan on switching the difficulty system to a modifier bases system. Modifers can be for example, high construction costs that make expansion more costly.

I also developed a simple plot system to allow players to occupy plots and build their city-states, making sure the camera follows the correct one.

Now, because Roblox is an online-only platform, the game engine also works on a client-server model, where the server handles validation and synchronization of game state, while client handles the user interface and input. This makes it essential to have a robust networking between the clients and server which is still a huge challenge for me. For example, I have to ensure that all players have the same game state at all times for plot ownership at all times, so that no player can claim a plot that is already occupied.

This is about as much content I have working at the moment. Due to lack of time, I have not been able to work on the game as much as I would have liked and due to O levels, development has been put on hold for the time being. However, I am still committed to continuing the development of Boundless Civitas and hope to release it in the future.

Choice of Tools

You may have noticed in the code snippets that they look familiar. Well, that's because I use VSCode to write my code and then sync it to Roblox Studio using their built-in file sync. This allows me to use a more powerful code editor with features like syntax highlighting, code completion, and version control. I also use GitHub to manage my code and track changes. I used to use Rojo for this sync but I found it too complex for my liking and I didn't like that I lost the ability to use the built-in Roblox Studio code editor. I also use Google Docs to plan out my game design to make sure the game doesn't undergo scope creep. A snippet of that is shwown below.

FAQs:

Q: When will Boundless Civitas be released?

A: I don't have a specific release date yet, because frankly speaking, even I don't know when it will be ready, though don't expect a release until next year at the minimum.

Q: Can I contribute to the development of Boundless Civitas?

A: You can contribute by providing feedback or even helping with development if you have the necessary skills, however I am not in the position to pay anyone and don't expect anyone else to join developing this project with me.

Q: Will Boundless Civitas be free to play?

A: Yes, Boundless Civitas will be free to play, but I may include optional in-game purchases to support the development of the game.

Q: What devices will Boundless Civitas be available on?

A: Boundless Civitas will be available on PC, mobile devices, and consoles, only on Roblox once it is released.

Q: Where did the name Boundless Civitas come from?

A: The name Boundless Civitas is a play on words, combining "boundless" (meaning unlimited or endless) with "civitas" (Latin for "city-state"). It represents the idea of an infinite, ever-expanding city-state where players can build and manage their own civilization.

Q: Can you help me with developing a roblox game?

A: I can provide some guidance and advice, but if you're asking for direct development assistance, the answer is a strong & firm no unless I get properly compensated.

Conclusion

To summarise, Boundless Civitas is a city-state building and management game in active development that aims to provide players with a unique and engaging experience. With its dynamic economy, simulated relations between other city-states, and AI NPCs, players will have the opportunity to build and manage their own city-state while facing the challenges and rewards of managing a complex system. I am excited to continue developing Boundless Civitas and look forward to sharing it with players in the future.
